# Summary Format

Per-author reports must be Markdown.

## Required sections

1. `总观点`
2. `分观点`

## `总观点`

Answer:

- what the author most clearly and consistently argued that day

Keep:

- one concise paragraph
- viewpoint consolidation only
- no extra background expansion

## `分观点`

Answer:

- the author's main sub-points for that day

Keep:

- `3-7` numbered points by default
- one clear judgment per point
- optional `证据：...` only when the point would otherwise be ambiguous

## Do Not Output

- `核心内容`
- `背景语境`
- `Spec相关`
- trade advice
- keyword statistics
- cross-author combined summary sections

## Current output scope

The current workflow requires:

- one `summary.md` per author under `{yyyymmdd}/{author}/`
